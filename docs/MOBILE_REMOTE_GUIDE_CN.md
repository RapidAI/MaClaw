# 手机远控使用说明

本文面向本程序中的“手机远控”功能，说明如何把桌面端正在运行的远程会话同步到 Hub，并在手机上通过 PWA 进行查看和控制。

## 1. 功能说明

手机远控由四部分组成：

- 桌面端 AICoder / MaClaw：负责启动远程会话、采集状态、同步到 Hub。
- Hub：负责机器注册、会话状态存储、PWA 页面和控制接口。
- Hub Center：可选，用于多 Hub 场景下的入口发现与管理。
- 手机端 PWA：在浏览器中打开的远控页面，用于查看会话、发送指令、执行中断等操作。

## 2. 使用前准备

在本机调试时，建议先启动本项目自带的本地远控栈：

1. 运行 `run_remote_stack.cmd`
2. 运行 `setup_remote_stack.cmd`
3. 确认以下地址可访问：
   - Hub Center 管理端：`http://127.0.0.1:9388/admin`
   - Hub 管理端：`http://127.0.0.1:9399/admin`
   - Hub PWA：`http://127.0.0.1:9399/app`

如果只是想快速验证一遍整条链路，可以直接运行：

- `run_full_remote_demo.cmd`

## 3. 桌面端设置

打开程序后，进入“设置”中的远控区域，按顺序填写和检查：

1. 打开“启用远程启动路径”
2. `Hub URL`
   - 本地调试可填 `http://127.0.0.1:9399`
3. `Hub Center URL`
   - 本地调试可填 `http://127.0.0.1:9388`
4. `远程邮箱`
   - 建议填写你在手机上也能访问确认链接的邮箱
5. 选择要远控的工具
   - 当前界面支持 `Claude`、`Codex`、`OpenCode`、`iFlow`、`Kilo`、`Kode`

然后建议依次点击：

1. `激活远程控制`
2. `运行就绪检查`
3. `运行 ConPTY 检测`
4. `运行启动探测`
5. 确认状态卡片显示正常后，点击 `启动远程 xxx`

## 4. 手机端登录方式

桌面端激活成功并启动远程会话后，可以在手机浏览器打开 Hub PWA：

- 本地调试地址：`http://127.0.0.1:9399/app`

如果你已经完成演示脚本，通常还可以直接使用自动登录风格地址：

- `http://127.0.0.1:9399/app?email=you@example.com&entry=app&autologin=1`

手机端登录流程通常为：

1. 输入邮箱，或通过带 `email` 参数的链接进入
2. Hub 发送登录确认链接
3. 在手机上打开确认链接
4. PWA 获取 viewer token 后进入远控页面

本地调试未接 SMTP 时，系统可能会直接返回一个可点击的确认链接。

## 5. 手机上能做什么

手机远控目前适合“轻控制”和“状态跟进”，不等同于完整桌面 IDE。

你可以在手机端进行：

- 查看当前在线机器和远程会话
- 查看当前任务、最近结果、进度摘要
- 查看重要事件和最近活动
- 向会话发送一条新的文字指令
- 对会话执行 `Interrupt`
- 对会话执行 `Kill`

桌面端设置页中的“受管远程会话”面板，展示的就是会同步到手机端的一组核心状态。

## 6. 推荐使用流程

推荐按下面的节奏使用：

1. 在电脑上配置好模型和项目
2. 在电脑上启动远程会话
3. 在手机上打开 PWA
4. 先看状态摘要和最近活动，确认会话没卡住
5. 再发送短指令，例如：
   - “先总结当前进度”
   - “继续执行上一步计划”
   - “停止当前任务并给出原因”
6. 如果会话异常：
   - 先用 `Interrupt`
   - 无响应时再用 `Kill`

## 7. 常见问题

### 7.1 手机上看不到会话

优先检查：

- 桌面端是否已成功“激活远程控制”
- `Hub URL` 是否可达
- 当前远程会话是否已成功启动
- Hub 中是否能看到机器和会话

可以运行：

- `remote_stack_status.cmd`
- `show_last_remote_demo.cmd`

### 7.2 手机能登录，但没有机器

通常说明：

- PWA 登录成功了
- 但桌面端还没有把机器/会话同步到 Hub

这时请在桌面端重新执行：

1. `刷新`
2. `运行就绪检查`
3. `启动远程工具`

## 7.3 发指令没有反应

优先检查：

- 远程工具是否仍在运行
- Hub 是否仍在线
- 桌面端会话状态是否已变成 `waiting_input`、`running` 或错误状态

必要时先尝试：

- `Interrupt`
- 然后重新发送更短的指令

## 8. 建议

- 手机端更适合看状态、发短命令，不建议长篇连续交互
- 首次部署建议先用本地 `9399/9388` 端口把整条链路跑通
- 如果要对外网开放，建议单独部署正式 Hub，并配置 HTTPS、邮箱发送和管理员账号
