# MaClaw Android Shell

This Android shell project was generated by MaClaw's Go-based mobile shell generator.

## Defaults

- App name: MaClaw
- Application ID: tech.rapidai.hubshell
- Start URL: file:///android_asset/bootstrap.html?hubcenter=http%3A%2F%2Fhubs.mypapers.top%3A9388
- Hub Center: http://hubs.mypapers.top:9388

## Output

All Android packaging artifacts are copied into `../dist/`.

Current filenames:

- Debug APK: `maclaw-debug.apk`
- Release APK: `maclaw-release.apk`
- Release AAB: `maclaw-release.aab`

## Scripts

- `build_android.cmd`
  Builds the debug APK.
- `build_android_release.cmd`
  Builds an unsigned release APK.
- `build_android_signed_release.cmd`
  Builds a signed release APK using `signing.env.cmd`.
- `build_android_aab.cmd`
  Builds an unsigned release AAB.
- `build_android_signed_aab.cmd`
  Builds a signed release AAB using `signing.env.cmd`.
- `create_android_keystore.cmd`
  Creates a new local keystore, defaulting to `maclaw-release.jks`.
- `signing.example.cmd`
  Template for signing configuration.
- `signing.env.cmd`
  Local signing configuration used by the signed release scripts.

## Common Flows

Debug APK:

```cmd
build_android.cmd
```

Unsigned release APK:

```cmd
build_android_release.cmd
```

Signed release APK:

```cmd
build_android_signed_release.cmd
```

Signed release AAB:

```cmd
build_android_signed_aab.cmd
```

## Signing Setup

If you do not already have a keystore:

```cmd
create_android_keystore.cmd
```

Then update `signing.env.cmd` with:

- `RELEASE_STORE_FILE`
- `RELEASE_STORE_PASSWORD`
- `RELEASE_KEY_ALIAS`
- `RELEASE_KEY_PASSWORD`

The current project already includes:

- `maclaw-release.jks`
- `signing.env.cmd`

So signed release packaging can run immediately on this machine.

## Android Studio

You can also open this folder in Android Studio and let Gradle sync normally. The command-line scripts are only wrappers around the same Gradle tasks and keep outputs collected under `mobile/dist/`.
